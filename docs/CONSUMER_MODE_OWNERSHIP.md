# Consumer Mode generated-file ownership

Consumer Mode is a scaffold, not a remote controller for downstream projects.

## Core rule

Every file written into a generated project becomes **project-owned immediately**.

The generator records provenance in `syntax.project.json`, including:

- the Syntax version;
- the recipe and recipe version;
- the selected visual direction and features;
- a deterministic configuration hash;
- the exact generated file list.

This record explains where the starting files came from. It does not grant Syntax permission to replace them later.

## Initial generation

The deterministic generator performs a complete preflight before writing files. If any destination file already exists, generation stops and lists the collisions. No project-owned file is silently replaced.

An overwrite option exists only for automated tests. It is unavailable during ordinary command execution.

Interactive setup will later add a separate, explicit template-conversion flow. That flow must identify known untouched Syntax template files before replacing them; it may not treat arbitrary files as disposable.

## Recipe versions

A recipe version records provenance. It is not a live dependency.

Updating a recipe in the Syntax repository does not rewrite projects generated by an older recipe. Safe migrations are deferred until ownership boundaries and narrowly scoped patch behavior are proven across real projects.

## Additive commands

Future `syntax:add` commands may:

- create new files after previewing the operation;
- update `syntax.project.json`;
- make narrow, clearly described edits required to load the selected feature.

They must not replace whole project-owned files simply because those files originally came from a recipe.

## Removal and Ship mode

Destructive cleanup is reserved for explicit Ship mode operations. Every deletion must be previewed, limited to known generated framework/demo material, and confirmed before execution.

## Framework boundary

All Consumer Mode code lives under `consumer/` or development scripts. It is excluded from `dist/syntax.css` and `dist/syntax.js`. Generated sites do not need the generator at runtime.
